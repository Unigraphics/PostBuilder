Program Start Sequence

This sequence has only one marker, the start of program.  This window allows you to determine which blocks are output when this event occurs in the tool path.  It is important to note that the start of program event occurs before any other event is handled.  Any UDE (User Defined Event) data is only available after the Start of Program Event.

Operation Start Sequence

This sequence defines the blocks that can be output at the start of each operation.  Every operation will have a first tool change, automatic tool change, manual tool change or no tool change.  See the list of scenarios below for the events output with each scenario.

The Operation Start Sequence has the following markers:

Start of Operation

This event is generated by the system and will be generated when you postprocess an operation.  This event will occur whether or not there is a tool change.

From Move

This event will only occur if there is a FROM programmed in the operation.  If you attach a block to this marker, it will only be output if there is a FROM in the operation.

First Tool

This event will only occur for the first tool change in a program.  This is useful if you already have a tool in the spindle and you wish to handle it differently.  If this marker has no blocks attached to it, the blocks being output will be the same as the automatic tool change block.

Automatic Tool Change

This event will occur on only if the operation contains a tool change.  A tool change is triggered by the system only when the tool is different between the current operation and the previous operation.  Note that this event is not related to the post command LOAD/TOOL.  This event will occur BEFORE the post command LOAD/TOOL is output.  You cannot use any of the LOAD/TOOL data with this event.

Manual Tool Change

This event will occur only if there is a manual tool change.

Initial Move

This event will be generated only if the operation has a tool change.  The system will create this event if there is no positioning move after the tool change.  All Start Post User Defined Events will be triggered before the initial or first move in an operation.  This event is the best place to handle the data triggered by operation level UDEs.

First Move

This event will only occur if there is no tool change in the operation.  Every operation will have either a first move or an initial move.  These events will allow you to combine blocks either before with or after the first motion block of the operation.  If there is no motion before the canned cycle, a motion block will be created to position to the rapid clearance plane above the hole.

Approach Move

This event will only occur if there is an approach move programmed in the operation.  Events attached to this event will only be output if there is an approach move in the operation.  Any blocks attached to this event will be output before the actual approach move motion block.

Engage Move

This event will only occur if there is an engage move in the operation.  Events attached to this event will only be output if there is an engage move in the operation.  Any blocks attached to this event will be output before the actual engage move motion block.

First Cut

This event will occur before the first cut move.  Any blocks attached to this event will be output before the actual first cut move motion block.
